# 平台自动登录 Skill

帮助用户登录跨境电商相关平台，保持登录状态以便进行数据采集和分析。

## 支持的平台

| 平台 | 登录方式 | 所需信息 |
|------|----------|----------|
| Temu | 手机号/邮箱 | 手机号或邮箱 + 密码 |
| Amazon | 邮箱 | 邮箱 + 密码 |
| 1688 | 淘宝/支付宝 | 手机号 + 密码（或扫码） |

## 登录信息配置

用户需要在 `~/.openclaw/workspace/TOOLS.md` 中配置登录信息：

```markdown
### 跨境平台账号

#### Temu
- 登录方式：邮箱
- 邮箱：your-email@example.com
- 密码：（运行时输入，不要写在这里）

#### Amazon (美国站)
- 登录方式：邮箱
- 邮箱：your-email@example.com
- 密码：（运行时输入，不要写在这里）

#### 1688
- 登录方式：扫码（推荐）或手机号
- 手机号：138xxxxxxxx
- 密码：（运行时输入，不要写在这里）
```

## 登录流程

### Temu 登录

1. 打开浏览器访问 https://www.temu.com
2. 点击登录按钮
3. 选择邮箱/手机号登录方式
4. 输入用户提供的邮箱/手机号
5. 输入密码
6. 处理可能的验证码（需要用户协助）
7. 确认登录成功

**验证登录成功**：页面显示用户头像或账户信息

### Amazon 登录

1. 打开浏览器访问 https://www.amazon.com
2. 点击 "Sign in" 按钮
3. 输入用户提供的邮箱
4. 点击继续
5. 输入密码
6. 处理可能的两步验证（需要用户协助）
7. 确认登录成功

**验证登录成功**：页面显示 "Hello, [用户名]"

### 1688 登录

1. 打开浏览器访问 https://www.1688.com
2. 点击登录按钮
3. 选择登录方式：
   - **扫码登录（推荐）**：显示二维码，用户用淘宝/支付宝 App 扫码
   - **密码登录**：输入手机号和密码
4. 处理滑块验证（如有）
5. 确认登录成功

**验证登录成功**：页面显示用户昵称

## 使用示例

### 示例 1：登录单个平台

**用户说：**
> 帮我登录 Temu

**AI 执行：**
1. 询问登录方式（邮箱/手机号）
2. 询问邮箱/手机号
3. 打开浏览器访问 Temu
4. 执行登录流程
5. 遇到密码输入时，提示用户输入密码
6. 遇到验证码时，提示用户协助完成
7. 确认登录成功

### 示例 2：登录所有平台

**用户说：**
> 帮我登录所有跨境平台

**AI 执行：**
1. 依次登录 Temu、Amazon、1688
2. 每个平台登录时询问必要信息
3. 报告登录状态

### 示例 3：检查登录状态

**用户说：**
> 检查一下各平台的登录状态

**AI 执行：**
1. 依次访问各平台
2. 检查是否已登录
3. 报告各平台登录状态

## 安全注意事项

1. **密码不存储**：密码仅在登录时临时使用，不会保存到任何文件
2. **使用独立浏览器配置**：使用 `profile: "openclaw"` 浏览器配置，与用户日常浏览器隔离
3. **Session 保持**：登录成功后，浏览器会保持 Cookie，下次访问无需重新登录
4. **敏感操作确认**：涉及支付、修改账户等敏感操作时，会先询问用户确认

## 常见问题处理

### 验证码/滑块验证

如果遇到验证码或滑块验证：
1. 截图发送给用户
2. 提示用户手动完成验证
3. 等待用户确认后继续

### 两步验证 (2FA)

如果遇到两步验证：
1. 提示用户查看手机/邮箱获取验证码
2. 用户提供验证码后输入
3. 继续登录流程

### 登录失败

如果登录失败：
1. 检查用户名/密码是否正确
2. 检查是否触发风控（IP 异常、设备异常）
3. 建议用户手动登录一次后再尝试

## 工具依赖

- `browser` 工具：用于打开浏览器和执行页面操作
- 使用 `profile: "openclaw"` 配置，保持独立的登录状态
